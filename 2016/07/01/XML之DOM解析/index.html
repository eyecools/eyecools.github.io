XML之DOM解析

DOM解析

1 介绍

Document Object Model（文档对象模型），将整个XML文档读入，构建驻留内存的树结构，通过便利树结构的任意节点，读取它的属性和值
dom解析时需要将xml整体读入,并且将xml结构化成树状
使用时经过树状结构进行读取
内存并不会占用太多

Dom解析有一个比较好的第三方工具类,gData
GDatatXMLNode是谷歌推出的一个开源的xml解析类,是对里边写满了.dylib的分装,速度要比NSXMLParser快很多
用起来方便,容易理解,在配置过程比较麻烦

2 步骤

2.1 获取GDataXMLNode.h/m文件，将GDataXMLNode.m添加到所需工程
2.2 添加类库libxml2.tbd到工程中
2.3 在工程中的”Build setting”页面中找到“header search path”，添加/usr/include/libxml2

3 代码

获取文件路径
NSString *path = [[NSBundle mainBundle]pathForResource:@"XML" ofType:@"txt"];

    //2 以data类型进行解析
    NSData *data = [NSData dataWithContentsOfFile:path];

    //3创建 GDataXMLDocument对象，此时xml文档内的所有节点以树的形式存在于 GDataXMLDocument对象中
    GDataXMLDocument *xmlDocument = [[GDataXMLDocument alloc]initWithData:data options:0 error:nil];

//    4 获取xml文档的根节点，根节点里包含了XML文档中所有的内容

    GDataXMLElement *rootElememt = [xmlDocument rootElement];

    //5 创建解析出来的学生数组
    self.dataArray = [NSMutableArray array];

    //便利根节点《students》的所有有子节点《student》
    for (GDataXMLElement *subElement in rootElememt.children) {
        Student *stu = [Student new];
        //再便利子节点《student》下的子标签name，zi，say，，weapon，去除子标签内容，赋值给stu
        for (GDataXMLElement *contactElement in subElement.children) {
            //这里的name指name，zi，say等标签
            if ([contactElement.name isEqualToString:@"name"]) {
                stu.name = contactElement.stringValue;

            }else if ([contactElement.name isEqualToString:@"zi"]) {
                stu.zi = contactElement.stringValue;
            }else if ([contactElement.name isEqualToString:@"say"]) {
                stu.say = contactElement.stringValue;
            }else if ([contactElement.name isEqualToString:@"weapon"]) {
                stu.weapon = contactElement.stringValue;
            }
            [stu setValue:contactElement.stringValue forKeyPath:contactElement.name];
